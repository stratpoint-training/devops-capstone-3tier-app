Course Overview

In Phase 1 you built a complete DevOps pipeline. In Phase 2, you will secure it — adding automated security scanning, secrets management, policy enforcement, supply chain controls, and runtime threat detection. Security is not a separate step; it runs at every stage.

⚠️ Prerequisite: Your Phase 1 pipeline (K8s, Helm, GitLab CI, ArgoCD, Observability) must be working before starting this phase.

Security Controls: P0 vs P1

P0 Non-negotiable
  • ● No secrets in Git or environment variables
  • ● Images scanned for CVEs before deploy
  • ● Manifests scanned for misconfigurations
  • ● Signed images verified at admission
  • ● Containers must not run as root
P1 Strongly Recommended
  • ● SBOM generated for every build
  • ● Runtime threat detection (Falco)
  • ● Admission controllers for all policies
  • ● Security alerts in Grafana
  • ● Network policies between pods

Two-Week Implementation Plan

Week 1 Pipeline & Cluster Security

D1–2 Add Gitleaks, Semgrep, Trivy to GitLab CI. Trigger a finding intentionally, fix it.
D3–4 Install ESO + Vault. Migrate DB credentials to ExternalSecret. Verify app still works.
D5 Run Checkov locally. Fix all HIGH/CRITICAL findings. Add Checkov to pipeline.

Week 2 Advanced Controls & Runtime

D1–2 Install Kyverno. Apply 4 policies in Audit mode. Fix violations. Switch to Enforce.
D3–4 Add Syft + Cosign to CI. Install Falco. Write custom rules. Connect to Loki.
D5 End-to-end test. Simulate attack (shell in container). Demo Falco detecting it. Presentation prep.
